# Screex v0.4 — Audio Narration Design

**Date:** 2026-06-15
**Status:** Approved (pending spec review)
**Branch:** `improve/audio-narration` (off latest `master`)

## 0. Why

Screen recordings of tutorials, Loom walkthroughs, and demos are usually **narrated**, and the
narration carries the *why* that OCR + thumbnails can't (intent, explanation, names of things).
The `index` path is video-only today — it discards the audio. Adding timestamped speech-to-text,
merged into the index and interleaved into the transcript, makes every view (transcript, Q&A,
docs) substantially richer for narrated content.

Engine chosen: **faster-whisper** (Whisper via CTranslate2). Verified locally: pip-only (no
system ffmpeg), reads an `.mp4` directly, returns timestamped segments, CPU-capable. It is an
**optional** dependency so the base install stays light.

## 1. Goals
- `screex index --audio` adds a timestamped transcript (`narration`) to `index.json`.
- `screex transcript` interleaves narration with UI states, time-aligned.
- Off by default (no ASR cost unless requested); robust when there's no audio track.

## 2. Components

### 2.1 `screex/core/audio.py` (new)
- `NarrationSegment` is defined in `index.py` (schema, §2.2); `audio.py` imports it.
- `_get_model(model="base")`: lazily construct and cache `faster_whisper.WhisperModel(model,
  device="cpu", compute_type="int8")` per model name. If `faster_whisper` is not importable,
  raise `RuntimeError("audio transcription needs the optional dependency: pip install
  'screex[audio]'")`.
- `transcribe(path, model="base", language=None) -> list[NarrationSegment]`:
  - Get the model; call `model.transcribe(str(path), language=language)`.
  - Convert each segment to `NarrationSegment(start=round(s.start,3), end=round(s.end,3),
    text=s.text.strip())`, dropping empty-text segments.
  - Wrap the transcribe call in try/except: on **any** failure (no audio stream, decode error)
    return `[]` (a missing/empty soundtrack must not abort indexing). The *model-load* error
    (missing optional dep) still propagates from `_get_model` — that's a setup problem, surfaced
    only when `--audio` was explicitly requested.

### 2.2 `screex/core/index.py`
- New dataclass:
  ```python
  @dataclass
  class NarrationSegment:
      start: float
      end: float
      text: str
  ```
- `ScreenIndex` gains `narration: list = field(default_factory=list)` (list of
  `NarrationSegment`). Update `to_dict` (serialize via `asdict`) and `from_dict`
  (`[NarrationSegment(**n) for n in d.get("narration", [])]` — `.get` keeps old indexes
  loadable). `save`/`load` unchanged.

### 2.3 `screex/cli.py`
- `index(..., audio=False, whisper_model="base")`. After states are built (and before saving),
  if `audio`: `narration = audio_mod.transcribe(recording, model=whisper_model)` (import
  `screex.core.audio as audio_mod` lazily), wrapped so a transcription failure logs and yields
  `[]` rather than aborting. Pass `narration=narration` into `ScreenIndex(...)`. `_log` a line
  like `"index: transcribed N narration segments"`.
- Flags on the `index` subparser: `--audio` (`store_true`) and `--whisper-model`
  (`default="base"`). Dispatch passes `audio=args.audio, whisper_model=args.whisper_model`.
- `transcript` subparser: add `--audio` and `--whisper-model`; when building the index, pass
  them through.

### 2.4 `screex/transcript.py`
- `format_transcript` interleaves narration. For each state, after the `Appeared`/`Gone` lines,
  add a `🗣 said: <text>` line built from `narration` segments overlapping `[t_start, t_end]`
  (overlap = `seg.start < state.t_end and seg.end > state.t_start`), joined in time order. Omit
  the line when no narration overlaps. A `ScreenIndex` with empty `narration` renders exactly as
  today (no behavior change).

## 3. Data flow
```
recording ─┬─ frames → segment → states ──────────────┐
           └─ (--audio) audio.transcribe → narration ──┤→ ScreenIndex(states, narration) → index.json
                                                         ↓
                       transcript: per state, UI changes + 🗣 narration spoken during it
```

## 4. Scope

### In
- `audio.py` (`transcribe`, lazy cached model, graceful no-audio, clear missing-dep error).
- `NarrationSegment` + `ScreenIndex.narration` (backward-compatible).
- `--audio` / `--whisper-model` on `index` and `transcript`; narration populated when requested.
- `format_transcript` narration interleaving.
- `screex[audio]` optional extra (faster-whisper) in `pyproject.toml`.
- README + SKILL.md notes; bump to 0.4.0.

### Out (YAGNI / later)
- Speaker diarization / multiple speakers.
- Word-level timestamps, audio events (music/clicks), translation.
- Per-state narration stored redundantly in the schema (derived at render time instead).
- GPU acceleration / cloud ASR.

## 5. Testing & success criteria
- **Schema:** `ScreenIndex` with `narration=[NarrationSegment(...)]` round-trips
  (save→load→equality); an old index dict without a `narration` key still loads (`[]`).
- **`format_transcript` interleaving:** a hand-built index with states + overlapping narration
  renders a `🗣 said:` line under the right state; a narration-empty index renders unchanged
  (golden assertions, no ASR).
- **`audio.transcribe` graceful failure:** monkeypatch `_get_model` to return a model whose
  `transcribe` raises → `transcribe()` returns `[]`.
- **Missing-dep error:** with `faster_whisper` un-importable, `_get_model` raises the clear
  `screex[audio]` message (simulated by monkeypatching the import).
- **Real ASR (integration, gated):** `pytest.importorskip("faster_whisper")`; transcribe a tiny
  known-speech clip and assert non-empty text — skipped in CI (base install has no `[audio]`).
- **CLI plumbing:** `--audio`/`--whisper-model` appear in `index --help`; `index(..., audio=False)`
  leaves `narration` empty (default path unchanged).
- Full suite + ruff stay green (`ruff check screex tests` before commits).

## 6. Environment / process
Windows 11, Python ≥3.9. `faster-whisper` optional (`pip install 'screex[audio]'`). Lands via
`improve/audio-narration` → PR/merge to `master`; publish 0.4.0 to PyPI after merge.
