# Squint v2.1 — Adaptive Cut Threshold Design Spec

**Date:** 2026-06-14
**Status:** Approved (pending spec review)
**Builds on:** Squint v2 event classification (`2026-06-14-squint-v2-event-classification-design.md`)

## 1. Problem

v2 classifies each event as `cut` or `motion` by comparing the grayscale-histogram
similarity at the event's `peak_frame` against a **fixed** threshold (`cut_threshold=0.5`).
A hand-labeled eval on two real clips showed the fixed threshold has a recall problem: it
misses real cuts whose transition similarity lands in the ~0.5–0.85 band.

- `whisper_crop.mp4` (2 known cuts): fixed-0.5 caught 1 of 2 (missed the dark→dark cut at
  frame 5, similarity ≈ 0.67).
- `cinemasings_output.mp4` (montage): every checked similarity dip was a genuine cut,
  including frame 41 at similarity ≈ 0.823, which fixed-0.5 misses.

In both clips, non-cut frames sit at similarity 0.99–1.0 and real cuts are clear outliers.
The signal is good; the **fixed threshold is mis-placed**.

A separate experiment confirmed a colour/HSV histogram is *worse*, not better (dark scenes
are near-black and desaturated, so hue/saturation carry less signal). Colour is not pursued.

## 2. Goal

Replace the fixed cut threshold with an **adaptive** one derived from each clip's own
similarity distribution, so cuts are detected as sharp drops relative to that clip's
baseline. Preserve a way to force an absolute threshold for backward compatibility.

## 3. Approach

A cut is a similarity that drops sharply below the clip's typical frame-to-frame
similarity:

- `baseline = median(similarities)` — robust to the rare low-similarity cut frames; ≈0.99
  on stable footage, lower on noisy clips, so it adapts per clip.
- `adaptive_threshold = clamp(baseline - margin, 0.0, 1.0)`, with `margin` default `0.15`
  (the value that caught both `whisper_crop` cuts and `cinemasings` frame 41 in the eval).
- An event is `cut` if `similarities[peak_frame] < threshold`, else `motion`.

### Rejected alternative (deferred)
A local-neighborhood baseline (compare each event to the median of nearby frames) is more
robust when a clip's baseline drifts mid-video, but more complex; the global median
sufficed on both eval clips. YAGNI for now.

## 4. Components

### 4.1 `squint/core/analyzer.py` — change `classify_events`

New signature (adds `margin`, makes `cut_threshold` optional):

```python
def classify_events(events, similarities, cut_threshold=None, margin=0.15):
    ...
```

Behavior:
- If `events` is empty or `similarities` is empty: return `events` unchanged (no median of
  an empty sequence).
- Determine the working threshold `T`:
  - If `cut_threshold is None` (adaptive): `T = min(1.0, max(0.0, median(similarities) - margin))`.
  - Else (absolute override): `T = cut_threshold`.
- For each event, with `s = similarities[event.peak_frame]`:
  - `s < T` → `type = "cut"`, `confidence = round((T - s) / T, 3)` if `T > 0` else `1.0`.
  - else → `type = "motion"`, `confidence = round((s - T) / (1 - T), 3)` if `T < 1` else `1.0`.
- Mutates and returns the same `EventRecord` objects.

`median` is computed with `numpy` (`float(np.median(np.asarray(similarities)))`).

### 4.2 `squint/cli.py` — flags

- `analyze(...)` signature: change `cut_threshold=0.5` to `cut_threshold=None`, add
  `margin=0.15`. Pass both into `analyzer.classify_events(events, similarities,
  cut_threshold, margin)`.
- Argparse `analyze` subparser:
  - `--cut-threshold`: `type=float, default=None` — "absolute histogram-similarity cut
    threshold; omit for adaptive (median - margin)".
  - Add `--cut-margin`: `type=float, default=0.15` — "how far below the clip's median
    similarity counts as a cut (adaptive mode)".
- Dispatch passes `cut_threshold=args.cut_threshold, margin=args.cut_margin`.

### 4.3 `SKILL.md` — guidance

Note that cut detection is adaptive per clip by default; `--cut-margin` widens/narrows
sensitivity, and `--cut-threshold` forces an absolute cutoff. The existing low-confidence
caveat (treat low-confidence labels as uncertain, escalate the PNG) remains.

## 5. Data flow (unchanged except the classify step)

```
per frame i:  score_i, sim_i  (sim_0 = 1.0)         # unchanged
events = group_events(scores, times, flags)          # unchanged
events = classify_events(events, sims,               # adaptive threshold now
                         cut_threshold=None, margin=0.15)
```

## 6. Scope

### In
- Adaptive logic + `margin` in `classify_events`.
- `--cut-threshold` default → adaptive; new `--cut-margin` flag; `analyze` signature update.
- SKILL.md note.
- Unit tests + real-clip proof.

### Out (YAGNI / future)
- Colour/HSV histograms (experimentally refuted).
- Local-neighborhood/drift-aware baseline.
- Multi-clip automated eval harness (eval remains the manual `docs/eval.md` procedure).
- Any change to `histogram_similarity`, `motion_score`, grouping, mapper, source, or the
  manifest schema.

## 7. Testing & success criteria

- **`classify_events` adaptive:** given events whose `peak_frame` similarities are clear
  outliers below a high baseline (e.g. baseline ≈ 0.99, one event at 0.67, one at 0.49) and
  `cut_threshold=None, margin=0.15`, both are labeled `cut`; an event at the baseline (≈0.99)
  is `motion`.
- **Absolute override preserved:** passing `cut_threshold=0.5` reproduces v2 behavior (an
  event at similarity 0.49 → `cut`, at 0.67 → `motion`).
- **Empty-input guard:** `classify_events([], [])` and `classify_events(events, [])` return
  without error.
- **Confidence bounds:** every classified event has `0.0 <= confidence <= 1.0`.
- **CLI:** `--cut-margin` appears in `analyze --help`; produced manifests have every event
  `type in {"cut","motion"}`.
- **Real-clip proof (manual):** re-run `analyze` on `whisper_crop.mp4` with defaults; both
  known cuts (frames 5 and 11) are labeled `type="cut"` (the v2 miss is fixed).
