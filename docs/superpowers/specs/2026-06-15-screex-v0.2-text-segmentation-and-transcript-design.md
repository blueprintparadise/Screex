# Screex v0.2 — Text-Aware Segmentation + Transcript Command

**Date:** 2026-06-15
**Status:** Approved (pending spec review)
**Branch:** `improve/text-aware-segmentation` (off latest `master`)
**Builds on:** the merged "hardening" work (dedup, drift detection, progress, `--screen`
capture, JPEG keyframes, `--lang`, CI, eval harness, noise-tolerant OCR).

## 0. Why

The hardening PR added the ops/robustness layer but did not fix the issue that makes Screex
feel broken on real screencasts: **segmentation can't see discrete local UI changes.**
`segment_stream` starts a new state only when whole-frame motion (per-frame `step` or
cumulative `drift` from the anchor) crosses `--change-threshold` (default 0.04). A dialog or
a few lines of text is a tiny fraction of a mostly-static frame, so neither `step` nor
`drift` crosses the threshold and the whole recording collapses to **one state**
("treated as static"). Dedup can only *merge* over-segmentation — it cannot recover
*under*-segmentation.

Two changes make Screex genuinely pleasant:
1. **Text-aware segmentation** — start a new state when the on-screen *text* changes, using
   the existing `ocr.text_similarity`. This directly fixes the core bug.
2. **`screex transcript`** — a deterministic markdown transcript straight from the CLI, so
   the tool delivers value (and a demo) without an LLM in the loop.

## 1. Goals
- `screex index` produces correct multi-state indexes on real screencasts **by default**,
  without the user tuning a magic threshold.
- `screex transcript <recording>` emits a clean, timestamped markdown step list.

## 2. Approach (segmentation): text-keyed with a motion pre-filter

Walk sampled frames. A cheap `motion_score` vs the previous frame **skips OCR on frames that
are visually identical** (a static stretch). Any frame that changed at all is OCR'd, and a
**new state begins when its text set diverges** from the current state's
(`text_similarity < text_threshold`). Motion is only a cost optimization; **text is the
segmentation signal**, so even a one-line change starts a new state.

Rejected: OCR-every-frame (simplest but slowest); hybrid motion-OR-timer candidates
(over-engineered — the pre-filter already surfaces every changed frame).

## 3. Components

### 3.1 `screex/core/segment.py`
- Extend `Segment` with an optional field: `ocr_text: object = None` (motion `segment_stream`
  leaves it `None`; the text segmenter fills it). Existing `segment_stream` is unchanged.
- Add:
  ```python
  def segment_by_text(frames, text_fn, text_threshold=0.80, motion_epsilon=0.003):
      ...
  ```
  - `frames`: iterator of `(idx, t, bgr)` (from `source.iter_frames`).
  - `text_fn`: callable `(bgr) -> list[str]` injected by the caller (keeps `segment.py`
    decoupled from `ocr` and unit-testable with a fake function).
  - Yields one `Segment(idx, t_start, t_end, frame_bgr, ocr_text)` per text-distinct UI state.
  - **Algorithm:**
    - First frame: OCR it, open state 0 with that text as the state's current text.
    - Subsequent frame: if `motion_score(prev_gray, gray) < motion_epsilon`, treat as
      identical — extend the current state (update `last_t`/`last_bgr`), **skip OCR**.
    - Otherwise OCR the frame. If `text_similarity(state_text, new_text) < text_threshold`,
      **emit** the current state (settled keyframe = last frame before this one, `ocr_text =
      state_text`) and open a new state with `new_text`. Else stay in the state and update
      `state_text = new_text` (so the emitted text reflects the settled content).
    - Always update `prev_gray`, `last_t`, `last_bgr`.
    - At stream end, emit the final state.
  - Memory-bounded: holds only `prev_gray`, the current `state_text`, and the last frame.
  - **`motion_epsilon` is a performance filter only** — its job is to skip frames that are
    essentially pixel-identical (motion ≈ 0) so we don't OCR a static screen 100×. It must be
    kept *below* any real UI text change. Default `0.003`; lowering it costs extra OCR (on
    compression-noisy video) but never misses a change — text-keying still collapses
    same-text frames, so correctness is preserved regardless.

### 3.2 `screex/cli.py` — `index`
- New parameters: `text_threshold=0.80`, `fast=False`, `motion_epsilon=0.003`.
- Default (text mode): build states from
  `segment.segment_by_text(frames, lambda b: ocr.extract_text(b, lang), text_threshold,
  motion_epsilon)`; each segment already carries `ocr_text`, so the loop writes images +
  computes `text_diff` + builds `ScreenState` (the existing dedup pass stays as a harmless
  safety net).
- `--fast` (motion mode): the existing path — `segment.segment_stream(frames,
  change_threshold)` then OCR each keyframe. Preserved for speed on simple cases and back-compat.
- New CLI flags on the `index` subparser: `--text-threshold` (default 0.80), `--fast`
  (store_true), `--motion-epsilon` (default 0.003). `--change-threshold` remains (used by
  `--fast`).
- **Single-state hint:** if the result has exactly one state and `duration` > 3s, `_log` a
  hint suggesting a lower `--text-threshold` or `--fps` (not an error).

### 3.3 `screex/transcript.py` (new, pure)
```python
def format_transcript(screen_index) -> str
```
Renders a `ScreenIndex` to markdown:
```
# Transcript — <video>  (<m:ss>)

## <m:ss>–<m:ss>  ·  State <n>
<ocr_text joined into a short context line>
**Appeared:** <text_added joined>
**Gone:** <text_removed joined>      # line omitted when empty
```
Includes a private `_fmt_time(seconds) -> "m:ss"` helper. Pure function over the dataclass —
no IO, fully unit-testable.

### 3.4 `screex/cli.py` — `transcript` subcommand
`screex transcript <recording> [-o OUT.md] [--from-index INDEX.json] [--fps N]
[--text-threshold F] [--fast]`:
- If `--from-index` is given, `ScreenIndex.load` it; otherwise call `index(...)` (text mode by
  default, honoring `--fps`/`--text-threshold`/`--fast`) and load the produced `index.json`.
- `format_transcript(...)` → write to `-o` if given, else print to stdout.

## 4. Data flow
```
recording → sample frames → segment_by_text (motion pre-filter → OCR changed frames → split on text change)
          → per state: write keyframe+thumbnail, text_diff → index.json
                                                                  ↓
                                       screex transcript → format_transcript → steps.md
```

## 5. Scope

### In
- `Segment.ocr_text` + `segment_by_text` in `segment.py`.
- Text-mode default + `--fast`/`--text-threshold`/`--motion-epsilon` + single-state hint in
  `index`.
- `transcript.py` (`format_transcript`) + `transcript` CLI subcommand.
- Tests (incl. the regression below) + README updates (index flags, a Transcript section).

### Out (YAGNI / later)
- `doc` / `report` CLI commands (those stay LLM-driven via the skill).
- Friendly `--sensitivity`/preset names (keep the explicit thresholds for now).
- Cursor/click detection, audio, UI-element grounding.
- Changing the motion `segment_stream` or any hardening behavior already merged.

## 6. Testing & success criteria
- **`segment_by_text`** (fake `text_fn` mapping a frame's fill value → a label):
  - a stream whose text changes once → exactly 2 states with correct `t_start`/`t_end` and
    each segment's `ocr_text`;
  - identical consecutive frames do not create extra states and do **not** call `text_fn`
    (assert call count — proves the motion pre-filter).
- **`format_transcript`**: on a hand-built `ScreenIndex`, the markdown contains each state's
  `m:ss–m:ss` header and an `**Appeared:**` line; an empty `text_removed` omits the `Gone`
  line.
- **CLI `transcript`**: on the synthetic screencast fixture, writes a markdown file containing
  the on-screen text and state headers.
- **Regression (the core fix):** a new `subtle_screencast_video` fixture — mostly static, same
  background, only a small text region changes between two screens. The changed region must be
  sized so the inter-state whole-frame motion lands **between `motion_epsilon` (0.003) and the
  motion `--change-threshold` (0.04)** — i.e. big enough that the text segmenter OCRs it, small
  enough that motion mode misses it. (The plan measures the fixture's actual motion and adjusts
  text size if needed.) `index(subtle, defaults)` must yield **≥ 2 states**; a companion
  assertion that `index(subtle, fast=True)` yields **1 state** documents the old failure and
  the contrast.
- Full suite stays green (existing motion/segment/index/cli tests unchanged).
